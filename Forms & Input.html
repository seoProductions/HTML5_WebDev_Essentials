Creating a Form

A Form is a Container that host's user-input aswell as specific action to the backend of a website. Attributes of Form are:

  • action="": REQUIRED: URL where the info will be processed
  • method="":
    • get: Values are only being passed in to the server.
    • post: HTTP is used to send values. More secure. Much Longer. Suitable for handling Databases

Input TAG

Once inside the form tag, you can include all the buttons, boxes, text-input that you want.

We use the <input> tag for this. It is an empty element that takes in attributes

  • type="": What type of input to create
  • name="": Passed to the server. Acts as the "key" in a key-value pair scenario. This is so that the webserver can identity what each data is.
  • id="": ID to identify this specific HTML element
  • maxlength: optional, limit the # of characters the user can input.

Radio-Button

Continuing with the Input TAG, another possible value for the attribute type="" is radio

For a radio-button and alot of other types, we use these attributes.

  • name="": same. Just please keep this one constant across all buttons.
  • value="": the value to be passed to the server
  • checked="": the default-selection option

Drop-Down List

Very similar to Radiobutton. Usefull if you have a long list to choose from.

We can create a Drop-Down List Box using 2 HTML Tags and some attributes of our choosing

  • <select> : Drop-Down Container that holds options that the user can select from.
  • name="": (attribute) : the Key in the Key-Value pair sent to the Server
  • size="": (attribute) : how many options show at once.
  • multiple="": (attribute) : allow for multiple options to be selected at the same time.
  • <option> : use to specify individual options to choose from.
  • value="": (attribute) : the value in the Key-Value pair sent to the Server
  • selected="": (attribute) : select an option by default. The first option is always selected by default, unless specified
